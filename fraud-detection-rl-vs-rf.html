Reinforcement Learning vs Random Forest on IEEE-CIS — 140k Transactions

Author

Léo Didier

Published

The Question

What if, instead of training a classifier to label transactions, we trained an AI agent that learns to catch fraud through trial and error — rewarded for correct calls, punished for mistakes?

That’s the premise of this project: pitting a Reinforcement Learning agent (PPO) against a Random Forest on the same fraud detection task. The results reveal a fascinating tension between two fundamentally different approaches to the same problem.

NoteDataset

IEEE-CIS Fraud Detection — 590,540 transactions with 394 features, merged with identity data (41 features). After cleaning and feature selection: 140,855 transactions, 44 features. Fraud rate: ~3.5% — a heavily imbalanced dataset.

The IEEE-CIS dataset is massive. Transaction data (amounts, card info, addresses) is joined with identity data (device type, browser, screen resolution). Let’s look at the class imbalance — the core challenge of fraud detection.

Approach 1: Reinforcement Learning (PPO)

This is the experimental approach. Instead of a standard classifier, we frame fraud detection as a sequential decision problem: an agent observes transactions one at a time and decides whether to approve or reject each one.

Custom Gymnasium Environment

Code 
class FraudDetectionEnv(gym.Env):
    def __init__(self, data, episode_length=1000):
        super().__init__()
        self.data = data.reset_index(drop=True)
        self.features = data.drop('isFraud', axis=1).values
        self.labels = data['isFraud'].values
        self.episode_length = episode_length

        self.action_space = spaces.Discrete(2)       # 0=Approve, 1=Reject
        self.observation_space = spaces.Box(
            low=-np.inf, high=np.inf,
            shape=(self.features.shape[1],),          # 44 features
            dtype=np.float32
        )

    def step(self, action):
        true_label = self.labels[self.episode_indices[self.current_step]]
        # Reward logic: +1 for correct, -1 for wrong
        reward = 1.0 if action == true_label else -1.0
        self.current_step += 1
        done = self.current_step >= self.episode_length
        obs = self.features[self.episode_indices[self.current_step]] if not done \
              else np.zeros(self.features.shape[1], dtype=np.float32)
        return obs, reward, done, False, {}
TipDesign Choice

The agent sees 44 features per transaction and outputs a binary action: approve (0) or reject (1). It gets +1 for correct decisions and -1 for mistakes. Episodes are 1,000 transactions long, sampled from random starting points in the dataset.

The RL agent wins on fraud recall (70.9% vs 46.9%) — it catches more fraudsters. But the cost is catastrophic: blocking 60% of legitimate customers is a dealbreaker in production. No payment processor would accept that trade-off.

Random Forest wins on everything else — accuracy, precision, speed, and operational cost. With basic hyperparameter tuning or a more sophisticated ensemble, the gap would only widen.

Key Takeaways

  1. RL is not inherently suited for tabular classification. The sequential decision framing adds complexity without adding value when transactions are independent.

  2. The accuracy-recall trade-off is real. The RL agent essentially learned to be suspicious of everything — catching more fraud but at an unacceptable cost to legitimate users.

  3. 50,000 timesteps is likely insufficient. With more training and a shaped reward function (higher penalty for false positives), the RL agent could potentially find a better balance. This is an area for future exploration.

  4. Simple baselines are powerful. A Random Forest with default parameters and no tuning already provides a production-viable fraud detection system. Always benchmark against simple methods first.

ImportantThe Bigger Picture

This project isn’t about RL being “bad” — it’s about choosing the right tool for the job. RL shines in sequential decision problems where actions affect future states (games, robotics, trading). For independent tabular classification, gradient-boosted trees and random forests remain king.

## Approach 1: Reinforcement Learning (PPO)

This is the experimental approach. Instead of a standard classifier, we frame fraud detection as a **sequential decision problem**: an agent observes transactions one at a time and decides whether to approve or reject each one.

### Custom Gymnasium Environment

```{python}
#| eval: false
class FraudDetectionEnv(gym.Env):
    def __init__(self, data, episode_length=1000):
        super().__init__()
        self.data = data.reset_index(drop=True)
        self.features = data.drop('isFraud', axis=1).values
        self.labels = data['isFraud'].values
        self.episode_length = episode_length

        self.action_space = spaces.Discrete(2)       # 0=Approve, 1=Reject
        self.observation_space = spaces.Box(
            low=-np.inf, high=np.inf,
            shape=(self.features.shape[1],),          # 44 features
            dtype=np.float32
        )

    def step(self, action):
        true_label = self.labels[self.episode_indices[self.current_step]]
        # Reward logic: +1 for correct, -1 for wrong
        reward = 1.0 if action == true_label else -1.0
        self.current_step += 1
        done = self.current_step >= self.episode_length
        obs = self.features[self.episode_indices[self.current_step]] if not done \
              else np.zeros(self.features.shape[1], dtype=np.float32)
        return obs, reward, done, False, {}
```

The confusion matrices tell the full story:

### Key Takeaways

1. **RL is not inherently suited for tabular classification.** The sequential decision framing adds complexity without adding value when transactions are independent.

2. **The accuracy-recall trade-off is real.** The RL agent essentially learned to be suspicious of everything — catching more fraud but at an unacceptable cost to legitimate users.

3. **50,000 timesteps is likely insufficient.** With more training and a shaped reward function (higher penalty for false positives), the RL agent could potentially find a better balance. This is an area for future exploration.

4. **Simple baselines are powerful.** A Random Forest with default parameters and no tuning already provides a production-viable fraud detection system. Always benchmark against simple methods first.

::: {.callout-important title="The Bigger Picture"}
This project isn't about RL being "bad" — it's about **choosing the right tool for the job**. RL shines in sequential decision problems where actions affect future states (games, robotics, trading). For independent tabular classification, gradient-boosted trees and random forests remain king.
:::
